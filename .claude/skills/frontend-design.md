此技能指导创建独特、生产级的前端界面，避免泛化的「AI 风格」审美。实现真实可用的代码，对美学细节和创意选择给予极高关注。

用户提供前端需求：一个组件、页面、应用或界面。可能包含关于目的、受众或技术约束的背景信息。

## 设计思维

编码前，先理解上下文并确定**大胆的美学方向**：
- **目的**：这个界面解决什么问题？谁在使用它？
- **基调**：选择一个极端：极简暴力、最大化混沌、复古未来、有机/自然、奢华/精致、趣味/玩具感、编辑/杂志风、粗野主义/原始、装饰艺术/几何、柔和/马卡龙、工业/实用主义等。风格无穷无尽，从中汲取灵感，设计一个真正契合美学方向的作品。
- **约束**：技术要求（框架、性能、无障碍访问）。
- **差异化**：什么让它**令人难忘**？有什么是用户会记住的唯一亮点？

**关键**：选定清晰的概念方向，精准执行。大胆的极繁主义和精致的极简主义都可以奏效——关键在于有意为之，而非强度本身。

然后实现可用的代码（HTML/CSS/JS、React、Vue 等），要求：
- 生产级别，功能完整
- 视觉震撼，令人难忘
- 风格统一，有清晰的美学观点
- 每处细节都经过精心打磨

## 前端美学指南

关注以下方面：
- **字体排版**：选择美观、独特、有趣的字体。避免 Arial、Inter 等通用字体；选择有个性的字体提升界面气质；将一款有特色的展示字体与一款精致的正文字体配对。
- **色彩与主题**：坚定执行一套统一的美学。使用 CSS 变量保持一致性。主色调加锐利强调色，优于平淡均分的多色方案。
- **动效**：使用动画实现效果和微交互。HTML 优先使用纯 CSS 方案；React 可使用 Motion 库。聚焦高影响力时刻：一个精心编排的页面加载配合交错显示（animation-delay），比分散的微交互更令人愉悦。使用滚动触发和出人意料的 hover 状态。
- **空间构图**：意想不到的布局。不对称。叠加。斜线流动。打破网格的元素。慷慨的留白或受控的密度。
- **背景与视觉细节**：营造氛围和层次感，而非默认纯色背景。添加与整体美学相匹配的上下文效果和纹理。运用渐变网格、噪点纹理、几何图案、分层透明、戏剧性阴影、装饰性边框、自定义光标、颗粒感叠层等创意手法。

**绝对不要**使用千篇一律的 AI 生成风格：过度使用的字体（Inter、Roboto、Arial、系统字体）、俗套的配色方案（白底紫色渐变）、可预测的布局与组件模式，以及缺乏针对性特色的通用设计。

创意解读，做出意外的选择，让设计真正为场景量身定制。每个设计都不应相同。在明暗主题、不同字体、不同美学之间交替。**绝不**在不同设计之间收敛到相同选择（例如反复使用 Space Grotesk）。

**重要**：将实现复杂度与美学愿景匹配。极繁主义设计需要包含大量动画和效果的精巧代码；极简或精致设计需要克制、精准，以及对间距、排版和微妙细节的细心把握。优雅来自于对愿景的精准执行。

记住：Claude 具备出色的创意能力。不要保守，展示在跳出框架、全力投入独特愿景时真正能创造的东西。

## 项目专属设计系统（两平米软件库）

> 若当前任务属于**两平米软件库**（Electron + Vue 3 桌面应用），以下设计令牌优先于上述通用指南。

### 设计令牌

| 用途 | 值 |
|---|---|
| 页面背景色 | `#ede8dc` |
| 主色（按钮/强调/选中态） | `#c4883a`，hover `#e09050` |
| 卡片背景 | `#ffffff`，边框 `#e0d4c0` |
| 主文字 | `#3a3028` / `#2c2420` |
| 次级文字 | `#6b5e4e`，辅助说明 `#8a7a6a` |
| 表格行 | bg `#fff`，hover `#faf7f2`，表头 `#f5f0e8` |
| 圆角 | 输入框/按钮 `10px`，卡片 `12px` |
| 字体 | `'Microsoft YaHei UI', 'Microsoft YaHei', 'PingFang SC', sans-serif` |
| 滚动条 | `width: 4px`，透明轨道，主色滑块 |

### z-index 层级

| 层 | 值 |
|---|---|
| 强制更新遮罩 | 1000 |
| WindowControls | 1500 |
| el-dialog | 2000 |

### 组件规范

- **WindowControls**：所有页面必须引入。登录页默认，其他页传 `:confirm-close="true" confirm-text="确认退出两平米软件库？"`
- **图标**：导航栏用 PNG（`src/assets/icons/`），其他用 Element Plus 图标（`@element-plus/icons-vue`）
- **动态图片路径**：必须用 `import` 引入，不能在模板里写字符串 `@/...`
- **颜色**：不硬编码，使用 `src/styles/themes.css` 中的 CSS 变量

### Vue 文件结构规范

```
<script setup>
// 导入
// 响应式状态
// 计算属性
// 生命周期
// 方法（均需注释）
</script>

<template>...</template>

<style scoped>...</style>
```

### 常用样式模式

```css
/* 卡片 */
background: #ffffff;
border: 1px solid #e0d4c0;
border-radius: 12px;

/* 主按钮 */
background: #c4883a;
border-radius: 10px;
color: #fff;

/* 滚动条 */
::-webkit-scrollbar { width: 4px; }
::-webkit-scrollbar-track { background: transparent; }
::-webkit-scrollbar-thumb { background: #c4883a; border-radius: 2px; }

/* 表格 */
.el-table { --el-table-bg-color: #fff; --el-table-row-hover-bg-color: #faf7f2; --el-table-header-bg-color: #f5f0e8; }
```
